---
title: 'Add Custom Properties'

description: 'Add custom properties to a request for filtering and analytics'
---

## Overview

Attach custom key-value properties to requests after they have been logged. Useful for tagging requests with metadata like environment, version, or feature flags.

## Authentication

<ParamField header="Authorization" type="string" required>
  Your Helicone API key. Format: `Bearer YOUR_API_KEY`
</ParamField>

## Path Parameters

<ParamField path="requestId" type="string" required>
  The unique identifier of the request
</ParamField>

## Request Body

<ParamField body="key" type="string" required>
  Property name (e.g., "environment", "version")
</ParamField>

<ParamField body="value" type="string" required>
  Property value (e.g., "production", "v1.2.3")
</ParamField>

## Response

<ResponseField name="data" type="null">
  Returns null on success
</ResponseField>

<ResponseField name="error" type="string">
  Error message if request failed
</ResponseField>

<RequestExample>

```bash cURL
curl --request PUT \
  --url https://api.helicone.ai/v1/request/req_abc123/property \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "key": "environment",
    "value": "production"
  }'
```

```python Python
import requests

url = "https://api.helicone.ai/v1/request/req_abc123/property"
headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}
payload = {
    "key": "environment",
    "value": "production"
}

response = requests.put(url, json=payload, headers=headers)
```

```javascript JavaScript
const response = await fetch(
  'https://api.helicone.ai/v1/request/req_abc123/property',
  {
    method: 'PUT',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      key: 'environment',
      value: 'production',
    }),
  }
);
```

</RequestExample>

<ResponseExample>

```json 201 Success
{

```


</ResponseExample>

## Use Cases

- Tag requests by deployment environment (staging, production)
- Track feature flags or A/B test variants
- Associate requests with specific app versions
- Mark requests by customer tier or plan
- Add custom metadata for analytics and filtering
