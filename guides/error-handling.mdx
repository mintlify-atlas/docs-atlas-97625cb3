# Error Handling Strategies

Learn how to handle errors gracefully in LLM applications with retries, fallbacks, and monitoring.

## Common LLM Errors

### Rate Limit Errors (429)

**Cause:** Too many requests to the provider

**Solution:** Implement exponential backoff

```typescript
import OpenAI from "openai";

const client = new OpenAI({
  baseURL: "https://ai-gateway.helicone.ai",
  apiKey: process.env.HELICONE_API_KEY,
});

async function withRetry<T>(
  fn: () => Promise<T>,
  maxRetries = 3
): Promise<T> {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn();
    } catch (error: any) {
      if (error.status === 429 && i < maxRetries - 1) {
        // Exponential backoff: 1s, 2s, 4s
        const delay = Math.pow(2, i) * 1000;
        await new Promise(resolve => setTimeout(resolve, delay));
        continue;
      }
      throw error;
    }
  }
  throw new Error("Max retries exceeded");
}

// Usage
const response = await withRetry(() =>
  client.chat.completions.create({
    model: "gpt-4o",
    messages: [{ role: "user", content: prompt }],
  })
);
```

### Timeout Errors

**Cause:** Request takes too long

**Solution:** Set appropriate timeouts

```typescript
const controller = new AbortController();
const timeoutId = setTimeout(() => controller.abort(), 30000); // 30s timeout

try {
  const response = await client.chat.completions.create(
    {
      model: "gpt-4o",
      messages: [{ role: "user", content: prompt }],
    },
    {
      signal: controller.signal,
    }
  );
  clearTimeout(timeoutId);
  return response;
} catch (error: any) {
  clearTimeout(timeoutId);
  if (error.name === "AbortError") {
    console.error("Request timed out");
    // Try with faster model
    return await client.chat.completions.create({
      model: "gpt-4o-mini", // Faster fallback
      messages: [{ role: "user", content: prompt }],
    });
  }
  throw error;
}
```

### Content Filter Errors

**Cause:** Request or response violates content policy

**Solution:** Handle gracefully and provide user feedback

```typescript
try {
  const response = await client.chat.completions.create({
    model: "gpt-4o",
    messages: [{ role: "user", content: userInput }],
  });
  return response;
} catch (error: any) {
  if (error.status === 400 && error.message?.includes("content_filter")) {
    return {
      success: false,
      error: "Your message contains content that cannot be processed. Please rephrase.",
    };
  }
  throw error;
}
```

## Automatic Fallbacks

### Model Fallbacks

Use Helicone's AI Gateway for automatic fallbacks:

```typescript
const response = await client.chat.completions.create({
  model: "gpt-4o,claude-sonnet-4,gpt-4o-mini", // Try in order
  messages: [{ role: "user", content: prompt }],
  headers: {
    "Helicone-Retry-Enabled": "true",
  },
});

// Helicone automatically:
// 1. Tries GPT-4o first
// 2. Falls back to Claude if GPT-4o fails
// 3. Falls back to GPT-4o-mini if Claude fails
```

### Provider Fallbacks

### Custom Fallback Logic

```typescript
async function robustLLMCall(prompt: string) {
  const strategies = [
    // Strategy 1: Primary model
    { model: "gpt-4o", temperature: 0.7 },
    // Strategy 2: Cheaper alternative
    { model: "gpt-4o-mini", temperature: 0.7 },
    // Strategy 3: Different provider
    { model: "claude-sonnet-4", temperature: 0.7 },
    // Strategy 4: Simplified prompt with cheaper model
    { model: "gpt-4o-mini", temperature: 0, prompt: simplifyPrompt(prompt) },
  ];

  for (const strategy of strategies) {
    try {
      return await client.chat.completions.create({
        model: strategy.model,
        messages: [{ role: "user", content: strategy.prompt || prompt }],
        temperature: strategy.temperature,
      });
    } catch (error) {
      console.warn(`Strategy failed: ${strategy.model}`, error);
      // Try next strategy
    }
  }

  throw new Error("All fallback strategies failed");
}
```

## Circuit Breaker Pattern

Prevent cascading failures:

```typescript
class CircuitBreaker {
  private failures = 0;
  private lastFailureTime?: number;
  private state: "closed" | "open" | "half-open" = "closed";

  constructor(
    private threshold = 5,
    private timeout = 60000 // 1 minute
  ) {}

  async execute<T>(fn: () => Promise<T>): Promise<T> {
    if (this.state === "open") {
      if (Date.now() - this.lastFailureTime! > this.timeout) {
        this.state = "half-open";
      } else {
        throw new Error("Circuit breaker is open");
      }
    }

    try {
      const result = await fn();
      this.onSuccess();
      return result;
    } catch (error) {
      this.onFailure();
      throw error;
    }
  }

  private onSuccess() {
    this.failures = 0;
    this.state = "closed";
  }

  private onFailure() {
    this.failures++;
    this.lastFailureTime = Date.now();
    if (this.failures >= this.threshold) {
      this.state = "open";
    }
  }
}

// Usage
const circuitBreaker = new CircuitBreaker();

const response = await circuitBreaker.execute(() =>
  client.chat.completions.create({
    model: "gpt-4o",
    messages: [{ role: "user", content: prompt }],
  })
);
```

## Graceful Degradation

### Provide Fallback Responses

```typescript
async function chatWithFallback(message: string) {
  try {
    // Try AI response
    const response = await client.chat.completions.create({
      model: "gpt-4o",
      messages: [{ role: "user", content: message }],
    });
    return response.choices[0].message.content;
  } catch (error) {
    console.error("AI failed, using fallback", error);

    // Fallback to rule-based responses
    return getRuleBasedResponse(message) || 
      "I'm having trouble right now. Please try again later or contact support.";
  }
}

function getRuleBasedResponse(message: string): string | null {
  const lowerMessage = message.toLowerCase();

  if (lowerMessage.includes("pricing") || lowerMessage.includes("cost")) {
    return "You can view our pricing at https://example.com/pricing";
  }

  if (lowerMessage.includes("support") || lowerMessage.includes("help")) {
    return "Please contact support at support@example.com or visit our help center.";
  }

  return null;
}
```

### Cache Previous Responses

```typescript
const responseCache = new Map<string, string>();

async function cachedLLMCall(prompt: string) {
  try {
    const response = await client.chat.completions.create({
      model: "gpt-4o",
      messages: [{ role: "user", content: prompt }],
      headers: {
        "Helicone-Cache-Enabled": "true",
      },
    });

    const content = response.choices[0].message.content;
    responseCache.set(prompt, content); // Cache locally too
    return content;
  } catch (error) {
    // If error, try to return cached response
    const cached = responseCache.get(prompt);
    if (cached) {
      console.warn("Using cached response due to error");
      return cached;
    }
    throw error;
  }
}
```

## Error Categories

### Client Errors (4xx)

| Status | Cause | Solution |
|--------|-------|----------|
| 400 | Invalid request format | Validate input before sending |
| 401 | Invalid API key | Check authentication |
| 403 | Forbidden content | Review content policies |
| 404 | Model not found | Check model name |
| 429 | Rate limit | Implement retries |

### Server Errors (5xx)

| Status | Cause | Solution |
|--------|-------|----------|
| 500 | Internal server error | Retry with backoff |
| 502 | Bad gateway | Use fallback provider |
| 503 | Service unavailable | Enable circuit breaker |
| 504 | Gateway timeout | Reduce prompt size |
