# Tracking and Optimizing LLM Costs

Learn how to track LLM costs across models and users, set up budget alerts, and implement strategies to reduce your AI spending.

## Overview

Helicone automatically tracks costs for every LLM request across 300+ models and providers. This guide shows you how to monitor costs and implement optimization strategies.

## Automatic Cost Tracking

### How It Works

Helicone calculates costs based on:
- **Token usage**: Input and output tokens
- **Model pricing**: Real-time pricing from provider APIs
- **Additional features**: Caching, function calls, embeddings
- **Provider-specific pricing**: Regional variations and special pricing

<Note>
  Costs are calculated automatically for all requests through the AI Gateway or proxy. No additional configuration needed.
</Note>

### Query User Costs

Retrieve cost data per user via API:

```bash
curl -X POST https://api.helicone.ai/v1/user/metrics/query \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "filter": {
      "user_ids": ["user_123"]
    },
    "timeFilter": {
      "start": "2024-01-01T00:00:00Z",
      "end": "2024-01-31T23:59:59Z"
    }
  }'
```

Response includes:
```json
{
  "data": [
    {
      "user_id": "user_123",
      "total_cost": 15.42,
      "total_requests": 523,
      "total_tokens": 125000,
      "average_cost_per_request": 0.029
    }
  ]
}
```

## Cost Optimization Strategies

### 1. Model Selection

Choose cost-effective models for each task:

<AccordionGroup>
  <Accordion title="Use Smaller Models When Possible" icon="sparkles">
    Replace expensive models with cheaper alternatives:
    
    ```typescript
    // Before: $0.03 per 1K tokens
    model: "gpt-4o"
    
    // After: $0.0001 per 1K tokens (300x cheaper)
    model: "gpt-4o-mini"
    ```
    
    Use expensive models only for:
    - Complex reasoning tasks
    - Creative content generation
    - Critical decision-making
    
    Use cheaper models for:
    - Simple classification
    - Data extraction
    - Summarization
    - Embeddings
  </Accordion>
  
  <Accordion title="Implement Intelligent Routing" icon="route">
    Route requests to different models based on complexity:
    
    ```typescript
    async function smartRoute(prompt: string) {
      const complexity = analyzeComplexity(prompt);
      
      const model = complexity > 0.7
        ? "gpt-4o"           // Complex tasks
        : "gpt-4o-mini";     // Simple tasks
      
      return await client.chat.completions.create({
        model,
        messages: [{ role: "user", content: prompt }],
      });
    }
    ```
  </Accordion>
  
  <Accordion title="Use Provider Routing" icon="server">
    Route to cheaper providers for equivalent quality:
    
    ```typescript
    // Try multiple providers, cheapest first
    const response = await client.chat.completions.create({
      model: "llama-3.3-70b/together,gpt-4o",
      messages: [{ role: "user", content: prompt }],
    });
    ```
  </Accordion>
</AccordionGroup>

### 2. Caching

Reduce costs by caching repeated requests:

```typescript
const response = await client.chat.completions.create({
  model: "gpt-4o",
  messages: [{ role: "user", content: "What is 2+2?" }],
  headers: {
    "Helicone-Cache-Enabled": "true",
    "Cache-Control": "max-age=86400", // Cache for 24 hours
  },
});

// Subsequent identical requests return cached response
// Cost: $0 (100% savings)
```

<Tip>
  Caching works best for:
  - Static content (FAQs, documentation)
  - Embeddings for unchanged documents
  - Translation of fixed text
  - Classification of similar inputs
</Tip>

### 3. Prompt Optimization

Reduce token usage with efficient prompts:

<Tabs>
  <Tab title="Before">
    ```typescript
    // Verbose prompt: ~100 tokens
    const prompt = `
      I would like you to please analyze the following text
      and provide a detailed summary of the main points.
      Please make sure to include all important information
      and present it in a clear and organized manner.
      
      Text: ${longText}
    `;
    ```
  </Tab>
  <Tab title="After">
    ```typescript
    // Concise prompt: ~20 tokens (80% reduction)
    const prompt = `Summarize key points:\n\n${longText}`;
    ```
  </Tab>
</Tabs>

### 4. Token Limits

Set maximum token limits to prevent runaway costs:

```typescript
const response = await client.chat.completions.create({
  model: "gpt-4o",
  messages: [{ role: "user", content: prompt }],
  max_tokens: 500, // Limit output to 500 tokens
});
```

### 5. Streaming Optimization

Use streaming to provide faster UX and stop early:

```typescript
const stream = await client.chat.completions.create({
  model: "gpt-4o",
  messages: [{ role: "user", content: prompt }],
  stream: true,
});

for await (const chunk of stream) {
  const content = chunk.choices[0]?.delta?.content;
  
  // Stop streaming if we have enough content
  if (hasEnoughContent(content)) {
    break; // Saves tokens and cost
  }
}
```

### 6. Batch Processing

Use OpenAI's Batch API for 50% cost savings:

```typescript
// Submit batch job (50% cheaper)
const batch = await client.batches.create({
  input_file_id: fileId,
  endpoint: "/v1/chat/completions",
  completion_window: "24h",
});

// Process results when ready
const results = await client.batches.retrieve(batch.id);
```

### Webhook Example

Receive cost alerts via webhook:

```typescript
// Your webhook endpoint
app.post("/webhooks/helicone", (req, res) => {
  const alert = req.body;
  
  if (alert.type === "cost_threshold") {
    // Alert: Daily budget exceeded
    console.log(`Cost alert: $${alert.current_cost} / $${alert.threshold}`);
    
    // Take action:
    // - Notify admin
    // - Pause non-critical workloads
    // - Switch to cheaper models
    
    sendAdminNotification(alert);
  }
  
  res.status(200).send("OK");
});
```

## Cost Analysis

### Analyze Cost Trends

Query historical costs to identify trends:

```bash
curl -X POST https://api.helicone.ai/v1/request/query \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "filter": {
      "request_created_at": {
        "gte": "2024-01-01T00:00:00Z",
        "lt": "2024-02-01T00:00:00Z"
      }
    },
    "metrics": ["cost", "tokens"]
  }'
```

### Cost by Model

Identify which models are most expensive:
