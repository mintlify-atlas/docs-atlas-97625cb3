## Get Started in 2 Minutes

This guide will walk you through integrating Helicone into your application. Choose the integration method that best fits your needs:

1. **AI Gateway** - Access 100+ models with automatic logging (Recommended)
2. **Proxy Logging** - Log requests with your own API keys
3. **Async Logging** - No proxy, asynchronous logging
4. **Manual Logging** - Full control over logging

<Note>
**Prerequisites:** You'll need a Helicone account and API key. Sign up at [helicone.ai/signup](https://helicone.ai/signup).
</Note>

---

## Method 1: AI Gateway (Recommended)

The AI Gateway gives you access to 100+ AI models through a single API with automatic request logging, intelligent routing, and fallback support.

<Steps>
  <Step title="Get Your API Key">
    1. Sign up at [helicone.ai/signup](https://helicone.ai/signup)
    2. Get your API key from the [dashboard](https://us.helicone.ai/developer)
    3. Add credits at [helicone.ai/credits](https://us.helicone.ai/credits) (optional for BYOK)
    
    <Info>
    The AI Gateway supports both **PTB (Pass-Through Billing)** where Helicone manages keys, and **BYOK (Bring Your Own Key)** where you use your own provider keys.
    </Info>
  </Step>
  
  <Step title="Install SDK">
    Install the OpenAI SDK (or your preferred provider's SDK):
    
    ```bash
    npm install openai
    ```
    
    Or with Python:
    
    ```bash
    pip install openai
    ```
  </Step>
  
  <Step title="Update Your Code">
    Change the `baseURL` to point to Helicone's AI Gateway:
    
    <CodeGroup>
    ```typescript TypeScript
    import OpenAI from "openai";
    
    const client = new OpenAI({
      baseURL: "https://ai-gateway.helicone.ai",
      apiKey: process.env.HELICONE_API_KEY,
    });
    
    const response = await client.chat.completions.create({
      model: "gpt-4o-mini",  // or any model from https://helicone.ai/models
      messages: [{ role: "user", content: "Hello!" }]
    });
    
    console.log(response.choices[0].message.content);
    ```
    
    ```python Python
    from openai import OpenAI
    
    client = OpenAI(
        base_url="https://ai-gateway.helicone.ai",
        api_key=os.getenv("HELICONE_API_KEY")
    )
    
    response = client.chat.completions.create(
        model="gpt-4o-mini",  # or any model
        messages=[{"role": "user", "content": "Hello!"}]
    )
    
    print(response.choices[0].message.content)
    ```
    </CodeGroup>
  </Step>
  
  <Step title="View Your Logs">
    Open the [Helicone Dashboard](https://us.helicone.ai/dashboard) to see:
    - Real-time request logs
    - Cost and latency metrics
    - Token usage analytics
    - Error tracking
    
    <Tip>
    Your first requests appear instantly in the dashboard!
    </Tip>
  </Step>
</Steps>

### Using Multiple Providers

The AI Gateway supports 100+ models from different providers. Just change the model name:

```typescript
// OpenAI
model: "gpt-4o-mini"

// Anthropic
model: "claude-sonnet-4"

// Google
model: "gemini-2.0-flash"

// Multiple models with fallback (tries in order)
model: "gpt-4o,claude-sonnet-4,gemini-2.0-flash"
```

See the full list at [helicone.ai/models](https://helicone.ai/models).

### Advanced: Provider-Specific Routing

You can specify exactly which provider to use with the `model/provider` format:

```typescript
// Use Claude from Anthropic
model: "claude-3-7-sonnet-20250219/anthropic"

// Use Claude from AWS Bedrock
model: "us.anthropic.claude-3-7-sonnet-20250219-v1:0/bedrock"

// Fallback between providers
model: "claude-3-7-sonnet-20250219/anthropic,us.anthropic.claude-3-7-sonnet-20250219-v1:0/bedrock"
```

## Method 2: Proxy Logging (BYOK)

Use Helicone's proxy to log requests while keeping your own API keys.

<Steps>
  <Step title="Get Your API Keys">
    1. Get your Helicone API key from [helicone.ai/signup](https://helicone.ai/signup)
    2. Get your provider API key (e.g., OpenAI, Anthropic)
  </Step>
  
  <Step title="Update Base URL">
    Change the base URL to use Helicone's proxy:
    
    <CodeGroup>
    ```typescript OpenAI
    import OpenAI from "openai";
    
    const openai = new OpenAI({
      apiKey: process.env.OPENAI_API_KEY,
      baseURL: "https://oai.helicone.ai/v1",
      defaultHeaders: {
        "Helicone-Auth": `Bearer ${process.env.HELICONE_API_KEY}`,
      },
    });
    
    const response = await openai.chat.completions.create({
      model: "gpt-4",
      messages: [{ role: "user", content: "Hello!" }],
    });
    ```
    
    ```typescript Anthropic
    import Anthropic from "@anthropic-ai/sdk";
    
    const anthropic = new Anthropic({
      apiKey: process.env.ANTHROPIC_API_KEY,
      baseURL: "https://anthropic.helicone.ai",
      defaultHeaders: {
        "Helicone-Auth": `Bearer ${process.env.HELICONE_API_KEY}`,
      },
    });
    
    const response = await anthropic.messages.create({
      model: "claude-sonnet-4",
      max_tokens: 1024,
      messages: [{ role: "user", content: "Hello!" }],
    });
    ```
    
    ```python Python (OpenAI)
    from openai import OpenAI
    
    client = OpenAI(
        api_key=os.getenv("OPENAI_API_KEY"),
        base_url="https://oai.helicone.ai/v1",
        default_headers={
            "Helicone-Auth": f"Bearer {os.getenv('HELICONE_API_KEY')}"
        }
    )
    
    response = client.chat.completions.create(
        model="gpt-4",
        messages=[{"role": "user", "content": "Hello!"}]
    )
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Add Custom Headers (Optional)">
    Enhance your logs with custom metadata:
    
    ```typescript
    const response = await openai.chat.completions.create(
      {
        model: "gpt-4",
        messages: [{ role: "user", content: "Hello!" }],
      },
      {
        headers: {
          "Helicone-Property-Environment": "production",
          "Helicone-Property-User-Id": "user-123",
          "Helicone-Cache-Enabled": "true",
        },
      }
    );
    ```
    
    Available headers:
    - `Helicone-Property-*`: Custom properties for filtering
    - `Helicone-User-Id`: Track requests by user
    - `Helicone-Session-Id`: Group requests in sessions
    - `Helicone-Cache-Enabled`: Enable response caching
    - `Helicone-Retry-Enabled`: Enable automatic retries
  </Step>
</Steps>

## Method 3: Async Logging

Log requests asynchronously without using a proxy. Uses OpenLLMetry for standardized LLM telemetry.

<Steps>
  <Step title="Install the Package">
    ```bash
    npm install @helicone/helicone openai
    ```
  </Step>
  
  <Step title="Initialize Logger">
    ```typescript
    import { HeliconeAsyncLogger } from "@helicone/helicone";
    import { OpenAI } from "openai";
    
    const logger = new HeliconeAsyncLogger({
      apiKey: process.env.HELICONE_API_KEY,
      providers: {
        openAI: OpenAI,
      },
    });
    logger.init();
    
    const openai = new OpenAI({
      apiKey: process.env.OPENAI_API_KEY,
    });
    
    // Requests are automatically logged
    const response = await openai.chat.completions.create({
      model: "gpt-4",
      messages: [{ role: "user", content: "Hello!" }],
    });
    ```
  </Step>
  
  <Step title="Add Custom Properties">
    ```typescript
    const logger = new HeliconeAsyncLogger({
      apiKey: process.env.HELICONE_API_KEY,
      providers: { openAI: OpenAI },
      heliconeMeta: {
        custom_properties: {
          project: "my-app",
          environment: "production",
        },
        user_id: "user-123",
      },
    });
    ```
  </Step>
</Steps>

<Info>
**No Proxy Latency** - Async logging happens in the background without adding latency to your requests.
</Info>

## Method 4: Manual Logging

Full control over what and when to log. Useful for custom integrations or providers not yet supported.

<Steps>
  <Step title="Install the Package">
    ```bash
    npm install @helicone/helpers
    ```
  </Step>
  
  <Step title="Log Requests">
    ```typescript
    import { HeliconeManualLogger } from "@helicone/helpers";
    
    const logger = new HeliconeManualLogger({
      apiKey: process.env.HELICONE_API_KEY,
    });
    
    const requestBody = {
      messages: [{ role: "user", content: "Hello!" }],
      model: "grok-3",
      stream: false,
    };
    
    const response = await logger.logRequest(
      requestBody,
      async (resultRecorder) => {
        // Make your API call
        const res = await fetch("https://api.x.ai/v1/chat/completions", {
          method: "POST",
          headers: {
            "Content-Type": "application/json",
            Authorization: `Bearer ${process.env.XAI_API_KEY}`,
          },
          body: JSON.stringify(requestBody),
        });
        
        const resBody = await res.json();
        
        // Record the result
        resultRecorder.appendResults(resBody);
        return resBody;
      },
      {
        "Helicone-Session-Id": "my-session",
      }
    );
    ```
  </Step>
</Steps>

## Session Tracking

Track multi-step workflows and agent interactions with sessions:

```typescript
import { randomUUID } from "crypto";
import OpenAI from "openai";

const openai = new OpenAI({
  apiKey: process.env.OPENAI_API_KEY,
  baseURL: "https://oai.helicone.ai/v1",
  defaultHeaders: {
    "Helicone-Auth": `Bearer ${process.env.HELICONE_API_KEY}`,
  },
});

const sessionId = randomUUID();

// First request in session
await openai.chat.completions.create(
  {
    model: "gpt-4",
    messages: [{ role: "user", content: "Generate a course outline" }],
  },
  {
    headers: {
      "Helicone-Session-Id": sessionId,
      "Helicone-Session-Name": "Course Generation",
      "Helicone-Session-Path": "/outline",
    },
  }
);

// Second request in same session
await openai.chat.completions.create(
  {
    model: "gpt-4",
    messages: [{ role: "user", content: "Generate chapter details" }],
  },
  {
    headers: {
      "Helicone-Session-Id": sessionId,
      "Helicone-Session-Name": "Course Generation",
      "Helicone-Session-Path": "/outline/chapter-1",
    },
  }
);
```

The session will appear as a grouped trace in the Helicone dashboard with:
- Total session cost
- Aggregate latency
- Hierarchical request tree
- Custom session properties

## Verify Your Integration

After making your first request:

1. **Check the Dashboard**: Visit [us.helicone.ai/dashboard](https://us.helicone.ai/dashboard)
2. **View Requests**: Click on "Requests" to see all logged requests

## Troubleshooting

<AccordionGroup>
  <Accordion title="Requests not appearing in dashboard">
    1. Verify your Helicone API key is correct
    2. Check that you're using the correct base URL
    3. Ensure the `Helicone-Auth` header is set (for proxy logging)
    4. Check for network errors in your application logs
    5. Try making a request directly with curl to test connectivity
  </Accordion>
  
  <Accordion title="401 Unauthorized errors">
    - Double-check your Helicone API key
    - Ensure the `Helicone-Auth` header format is `Bearer <api-key>`
    - Verify your account is active at [helicone.ai](https://helicone.ai)
  </Accordion>
  
  <Accordion title="Provider API errors (OpenAI, Anthropic, etc.)">
    - Verify your provider API key is correct
    - Check that the model name is valid
    - Ensure you have credits/quota with the provider
    - For AI Gateway: verify you have Helicone credits for PTB
  </Accordion>
  
  <Accordion title="High latency">
    - Use Async Logging for zero proxy latency
    - Enable caching for frequently-used prompts
    - Check your network connection to Helicone
    - Use regional endpoints if available
  </Accordion>
</AccordionGroup>
