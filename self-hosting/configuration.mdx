Complete reference for configuring your self-hosted Helicone deployment. All services are configured through environment variables.

## Core Configuration

### Authentication & Security

<ParamField path="BETTER_AUTH_SECRET" type="string" required>
  Secret key for authentication. Must be at least 32 characters.
  
  ```bash
  BETTER_AUTH_SECRET="MKUcaeqyMD7UBkGeFYY5hwxKS1aB6Vsi"
  ```
  
  <Warning>
    **Always change this before production!** Generate a secure key:
    ```bash
    openssl rand -base64 32
    ```
  </Warning>
</ParamField>

<ParamField path="NEXT_PUBLIC_BETTER_AUTH" type="boolean" default="true">
  Enable Better Auth authentication system (recommended for self-hosting).
  
  ```bash
  NEXT_PUBLIC_BETTER_AUTH="true"
  ```
</ParamField>

### Database Configuration

<ParamField path="SUPABASE_DATABASE_URL" type="string" required>
  PostgreSQL connection string for the application database.
  
  ```bash
  # Docker Compose
  SUPABASE_DATABASE_URL="postgresql://postgres:testpassword@db:5432/helicone_test"
  
  # External PostgreSQL
  SUPABASE_DATABASE_URL="postgresql://user:pass@your-host:5432/helicone"
  ```
</ParamField>

<ParamField path="DATABASE_URL" type="string">
  Alternative PostgreSQL connection URL (for compatibility).
  
  ```bash
  DATABASE_URL="postgresql://postgres:postgres@localhost:54322/postgres"
  ```
</ParamField>

### ClickHouse Configuration

<ParamField path="CLICKHOUSE_CREDS" type="json" required>
  ClickHouse connection credentials (used by Jawn).
  
  ```bash
  CLICKHOUSE_CREDS='{
    "CLICKHOUSE_HOST": "http://clickhouse:8123",
    "CLICKHOUSE_USER": "default",
    "CLICKHOUSE_PASSWORD": "",
    "CLICKHOUSE_HQL_USER": "hql_user",
    "CLICKHOUSE_HQL_PASSWORD": ""
  }'
  ```
</ParamField>

<ParamField path="CLICKHOUSE_HOST" type="string" required>
  ClickHouse HTTP endpoint.
  
  ```bash
  # Internal
  CLICKHOUSE_HOST="http://clickhouse:8123"
  
  # External
  CLICKHOUSE_HOST="https://your-clickhouse-host.com:8443"
  ```
</ParamField>

<ParamField path="CLICKHOUSE_USER" type="string" default="default">
  ClickHouse username.
  
  ```bash
  CLICKHOUSE_USER="default"
  ```
</ParamField>

<ParamField path="CLICKHOUSE_PASSWORD" type="string">
  ClickHouse password (empty for local development).
  
  ```bash
  CLICKHOUSE_PASSWORD="your-secure-password"
  ```
</ParamField>

### Object Storage (S3/MinIO)

<ParamField path="S3_ENDPOINT" type="string" required>
  S3-compatible storage endpoint.
  
  ```bash
  # MinIO (local)
  S3_ENDPOINT="http://minio:9000"
  
  # AWS S3
  S3_ENDPOINT="https://s3.amazonaws.com"
  
  # Custom S3-compatible
  S3_ENDPOINT="https://s3.your-provider.com"
  ```
</ParamField>

<ParamField path="S3_ACCESS_KEY" type="string" required>
  S3 access key ID.
  
  ```bash
  # MinIO default
  S3_ACCESS_KEY="minioadmin"
  
  # AWS
  S3_ACCESS_KEY="AKIAIOSFODNN7EXAMPLE"
  ```
</ParamField>

<ParamField path="S3_SECRET_KEY" type="string" required>
  S3 secret access key.
  
  ```bash
  # MinIO default
  S3_SECRET_KEY="minioadmin"
  
  # AWS
  S3_SECRET_KEY="wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY"
  ```
</ParamField>

<ParamField path="S3_BUCKET_NAME" type="string" default="request-response-storage">
  Bucket name for storing request/response bodies.
  
  ```bash
  S3_BUCKET_NAME="helicone-requests"
  ```
</ParamField>

<ParamField path="S3_PROMPT_BUCKET_NAME" type="string" default="prompt-body-storage">
  Bucket name for storing prompt templates.
  
  ```bash
  S3_PROMPT_BUCKET_NAME="helicone-prompts"
  ```
</ParamField>

<ParamField path="S3_ENDPOINT_PUBLIC" type="string">
  Public-facing S3 endpoint for browser access.
  
  ```bash
  # For local MinIO
  S3_ENDPOINT_PUBLIC="http://localhost:9000"
  
  # For production
  S3_ENDPOINT_PUBLIC="https://s3.amazonaws.com"
  ```
</ParamField>

## Service Configuration

### Jawn (API Server)

<ParamField path="PORT" type="number" default="8585">
  Port for Jawn API server.
  
  ```bash
  PORT=8585
  ```
</ParamField>

<ParamField path="JAWN_PORT" type="number" default="8585">
  Docker Compose override for Jawn port.
  
  ```bash
  JAWN_PORT=8585
  ```
</ParamField>

<ParamField path="JAWN_PUBLIC_URL" type="string">
  Public URL for Jawn API (used by web frontend).
  
  ```bash
  # Development
  JAWN_PUBLIC_URL="http://localhost:8585"
  
  # Production
  JAWN_PUBLIC_URL="https://api.helicone.yourdomain.com"
  ```
</ParamField>

<ParamField path="VALHALLA_URL" type="string">
  Internal URL for Jawn service (used by workers).
  
  ```bash
  VALHALLA_URL="http://jawn:8585"
  ```
</ParamField>

### Web Dashboard

<ParamField path="NEXT_PUBLIC_HELICONE_JAWN_SERVICE" type="string" required>
  URL for Jawn API service (accessible from browser).
  
  ```bash
  # Development
  NEXT_PUBLIC_HELICONE_JAWN_SERVICE="http://localhost:8585"
  
  # Production
  NEXT_PUBLIC_HELICONE_JAWN_SERVICE="https://api.helicone.yourdomain.com"
  ```
</ParamField>

<ParamField path="NEXT_PUBLIC_APP_URL" type="string">
  Public URL of the web dashboard.
  
  ```bash
  NEXT_PUBLIC_APP_URL="https://helicone.yourdomain.com"
  ```
</ParamField>

<ParamField path="SITE_URL" type="string">
  Base site URL for authentication redirects.
  
  ```bash
  SITE_URL="http://localhost:3000"
  ```
</ParamField>

<ParamField path="NODE_ENV" type="string" default="development">
  Node.js environment mode.
  
  ```bash
  # Development
  NODE_ENV="development"
  
  # Production
  NODE_ENV="production"
  ```
</ParamField>

### Worker Services

<ParamField path="WORKER_TYPE" type="string">
  Type of worker to run. Options: `OPENAI_PROXY`, `ANTHROPIC_PROXY`, `HELICONE_API`, `GATEWAY_API`, `GENERATE_API`.
  
  ```bash
  WORKER_TYPE="OPENAI_PROXY"
  ```
</ParamField>

<ParamField path="WORKER_PORT" type="number">
  Port for worker service.
  
  ```bash
  # OpenAI Proxy
  WORKER_PORT=8787
  
  # Anthropic Proxy
  WORKER_PORT=8790
  ```
</ParamField>

<ParamField path="HELICONE_WORKER_URL" type="string">
  URL of the worker proxy service.
  
  ```bash
  # For Jawn (Docker Compose)
  HELICONE_WORKER_URL="http://localhost:8585/v1/gateway/oai"
  
  # For standalone worker
  HELICONE_WORKER_URL="http://localhost:8787"
  ```
</ParamField>

<ParamField path="GATEWAY_TARGET" type="string">
  Target URL for gateway worker.
  
  ```bash
  GATEWAY_TARGET="https://openrouter.ai"
  ```
</ParamField>

## Email Configuration

<ParamField path="SMTP_HOST" type="string">
  SMTP server hostname.
  
  ```bash
  # Development (MailHog)
  SMTP_HOST="mailhog"
  
  # Production (Gmail)
  SMTP_HOST="smtp.gmail.com"
  
  # Production (SendGrid)
  SMTP_HOST="smtp.sendgrid.net"
  ```
</ParamField>

<ParamField path="SMTP_PORT" type="number">
  SMTP server port.
  
  ```bash
  # Development
  SMTP_PORT=1025
  
  # Production (TLS)
  SMTP_PORT=587
  
  # Production (SSL)
  SMTP_PORT=465
  ```
</ParamField>

<ParamField path="SMTP_SECURE" type="boolean" default="false">
  Use SSL/TLS for SMTP.
  
  ```bash
  SMTP_SECURE="true"
  ```
</ParamField>

<ParamField path="SMTP_USER" type="string">
  SMTP username.
  
  ```bash
  SMTP_USER="your-email@gmail.com"
  ```
</ParamField>

<ParamField path="SMTP_PASS" type="string">
  SMTP password or app password.
  
  ```bash
  SMTP_PASS="your-app-password"
  ```
</ParamField>

<ParamField path="SMTP_ADMIN_EMAIL" type="string">
  Admin email address for system notifications.
  
  ```bash
  SMTP_ADMIN_EMAIL="admin@yourdomain.com"
  ```
</ParamField>

## Example Configurations

### Minimal Docker Compose

```bash
# .env
BETTER_AUTH_SECRET="$(openssl rand -base64 32)"
POSTGRES_PASSWORD="your-secure-password"
JAWN_PORT=8585
JAWN_PUBLIC_URL="http://localhost:8585"
VERCEL_ENV="development"

# MinIO (included in docker-compose)
S3_ENDPOINT="http://minio:9000"
S3_ACCESS_KEY="minioadmin"
S3_SECRET_KEY="minioadmin"
S3_BUCKET_NAME="request-response-storage"
S3_PROMPT_BUCKET_NAME="prompt-body-storage"
```

### Production with External Services

```bash
# .env
BETTER_AUTH_SECRET="$(openssl rand -base64 32)"
VERCEL_ENV="production"
NODE_ENV="production"

# Database

# ClickHouse
CLICKHOUSE_HOST="https://clickhouse.example.com:8443"
CLICKHOUSE_USER="helicone_user"
CLICKHOUSE_PASSWORD="SECURE_PASSWORD"
CLICKHOUSE_CREDS='{
  "CLICKHOUSE_HOST": "https://clickhouse.example.com:8443",
  "CLICKHOUSE_USER": "helicone_user",
  "CLICKHOUSE_PASSWORD": "SECURE_PASSWORD",
  "CLICKHOUSE_HQL_USER": "hql_user",
  "CLICKHOUSE_HQL_PASSWORD": "HQL_PASSWORD"

# AWS S3
S3_ENDPOINT="https://s3.us-east-1.amazonaws.com"

S3_BUCKET_NAME="helicone-prod-requests"
```

### Kubernetes Secrets

For Kubernetes, store sensitive values in secrets:

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: helicone-secrets
  namespace: helicone
type: Opaque
stringData:
  BETTER_AUTH_SECRET: "your-secret-key"
  POSTGRES_PASSWORD: "your-db-password"
  CLICKHOUSE_PASSWORD: "your-clickhouse-password"
  S3_ACCESS_KEY: "your-access-key"
  S3_SECRET_KEY: "your-secret-key"
  SMTP_PASS: "your-smtp-password"
```

## Security Best Practices

<AccordionGroup>
  <Accordion title="Generate Strong Secrets" icon="key">
    Always generate cryptographically secure secrets:
    
    ```bash
    # Generate auth secret
    openssl rand -base64 32
    
    # Generate database password
    openssl rand -base64 24
    
    # Generate UUID
    uuidgen
    ```
  </Accordion>

  <Accordion title="Use Environment-Specific Files" icon="file">
    Keep separate configurations:
    
    ```bash
    .env.development    # Local development
    .env.staging        # Staging environment
    .env.production     # Production (never commit!)
    ```
  </Accordion>

  <Accordion title="Rotate Credentials" icon="rotate">
    Regularly rotate sensitive credentials:
    
    - Database passwords: Every 90 days
    - API keys: Every 6 months
    - Auth secrets: Annually or after incidents
  </Accordion>

  <Accordion title="Use Secrets Managers" icon="vault">
    For production, use secrets management:
    
    - **AWS**: AWS Secrets Manager
    - **GCP**: Google Secret Manager
    - **Azure**: Azure Key Vault
    - **Kubernetes**: Sealed Secrets or External Secrets Operator
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Environment variables not loading" icon="circle-exclamation">
    Check `.env` file location and format:
    
    ```bash
    # Verify file exists
    ls -la .env
    
    # Check for syntax errors (no spaces around =)
    cat .env
    
    # Restart services to reload
    docker compose restart
    ```
  </Accordion>

  <Accordion title="Database connection fails" icon="database">
    Verify connection string format:
    
    ```bash
    # Correct format
    postgresql://[user]:[password]@[host]:[port]/[database]
    
    # Test connection
    psql "postgresql://user:pass@host:5432/db" -c "SELECT 1"
    ```
  </Accordion>

  <Accordion title="S3 storage errors" icon="box">
    Verify S3 credentials and bucket access:
    
    ```bash
    # Test with AWS CLI
    aws s3 ls s3://your-bucket --endpoint-url $S3_ENDPOINT
    
    # Check MinIO
    docker logs helicone-minio
    ```
  </Accordion>
</AccordionGroup>
