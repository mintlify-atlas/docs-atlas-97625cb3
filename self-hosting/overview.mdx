Helicone can be self-hosted in your own infrastructure, giving you complete control over your data and deployment environment. This is ideal for organizations with strict data governance requirements or those who want to customize their deployment.

## Architecture Components

Helicone consists of five core services:

<AccordionGroup>
  <Accordion title="Web - Frontend Platform" icon="desktop">
    The NextJS-based dashboard for visualizing requests, managing prompts, and analyzing your LLM usage. Provides the user interface for all Helicone features.
  </Accordion>

  <Accordion title="Jawn - API Server" icon="server">
    Express + Tsoa-based API server that handles log collection, request processing, and serves the REST API. This is the core backend service.
  </Accordion>

  <Accordion title="Worker - Proxy Service" icon="network-wired">
    Cloudflare Workers-based proxy that intercepts and logs LLM requests. Can run as OpenAI Proxy, Anthropic Proxy, or Helicone API.
  </Accordion>

  <Accordion title="PostgreSQL - Application Database" icon="database">
    Stores user data, authentication, organization settings, and application state. Can use your own PostgreSQL instance or Supabase.
  </Accordion>

  <Accordion title="ClickHouse - Analytics Database" icon="chart-line">
    High-performance columnar database for storing and analyzing request logs, metrics, and time-series data.
  </Accordion>

  <Accordion title="MinIO - Object Storage" icon="box">
    S3-compatible storage for request/response bodies and prompt storage. Can be replaced with AWS S3 or compatible services.
  </Accordion>
</AccordionGroup>

## Quick Start

The fastest way to try self-hosted Helicone is with Docker:

```bash
git clone https://github.com/Helicone/helicone.git
cd helicone/docker
cp .env.example .env
./helicone-compose.sh helicone up
```

Access the dashboard at `http://localhost:3000`

<Note>
  For production deployments, see our detailed [Docker](/self-hosting/docker) and [Kubernetes](/self-hosting/kubernetes) guides.
</Note>

## System Requirements

### Minimum (Development)
- **CPU**: 4 cores
- **RAM**: 8GB
- **Storage**: 20GB SSD
- **OS**: Linux, macOS, or Windows with Docker

### Recommended (Production)
- **CPU**: 8+ cores
- **RAM**: 16GB+
- **Storage**: 100GB+ SSD (for ClickHouse)
- **Network**: Low latency connection to your LLM providers

## Default Ports

When running all services:

| Service | Port | Description |
|---------|------|-------------|
| Web Dashboard | 3000 | NextJS frontend |
| Jawn API | 8585 | Backend API and LLM proxy |
| PostgreSQL | 54388 | Application database |
| ClickHouse HTTP | 18123 | Analytics database HTTP interface |
| ClickHouse Native | 19000 | Analytics database native interface |
| MinIO API | 9000 | S3-compatible storage |
| MinIO Console | 9001 | MinIO admin interface |
| MailHog SMTP | 1025 | Email testing (dev only) |
| MailHog Web | 8025 | Email testing UI (dev only) |
